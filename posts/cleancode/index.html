Clean code notes

2022-05-07

Names

Meaningful names increase readability.

Names that communicate intent increase readability

Functions

DO’s:

  • Function should do 1 thing.
  • If a function is too big, it should be extracted into a class.
  • few arguments as possible (no more than 3)

DONT’s

  • do not pass boolean as an argument
  • do not use output argument, use return statement
  • do not do defensive programming (check for null)

Command - Query separation

Function should either do something or return something not both. Command returns void or throw an exception and has side effects. Query returns a value and does not have a side effects.

Side effect - when a function changes the variable that outlives the function's call

Law of Demeter

Module should have knowledge only about immediate collaborators. An example of violation Law of Demeter would be:

transaction.getCustomer().getAddress();

Transaction isn’t supposed to have knowledge about address.

Error handling

DO’s:

DON’Ts

  • don’t pass null
  • don’t return null
